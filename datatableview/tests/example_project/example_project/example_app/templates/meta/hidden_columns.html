hidden_columns

Columns from demo: n pingbacks (not shown)

A list of column names that should be hidden from the user on the frontend.

The use case for this option is that a column might need to be generated and communicated during AJAX requests, but stripped out for that specific page. This might allow for reusable data sources where the page is modifying what is being seen. Because new data sources are easy to make, however, we recommend avoiding too much funny business.

The "hidden" data is still going across the wire anyway, and is inspectable by prying eyes. Don't rely on this for sensitive data.

Another use for this feature in dataTables.js is for plugins. Exporters like the CSV one have access to the "hidden" fields and can export them to a file even if they are not visible on the web page.
